# ZPP011 表格组件选型对比

| 维度 | pandastable | PyQt5 QTableView | 优化现有 Treeview |
|------|------------|-----------------|------------------|
| **性能（万行）** | ⚠️ 中等（tkinter Canvas瓶颈，5000行可接受，万行可能卡顿） | ✅ 优秀（QAbstractTableModel虚拟滚动，10万行无压力） | ⚠️ 较差（tkinter Treeview 3000+行开始卡顿） |
| **颜色标记** | ✅ 支持（dfstyle/单元格回调，可自定义） | ✅ 优秀（QItemDelegate/QStyledItemDelegate，完全控制） | ✅ 已实现（tags+tag_configure，但粒度有限） |
| **业务主键映射** | ✅ 可实现（DataFrame索引绑定，排序后仍准确） | ✅ 优秀（model.row()天然支持） | ⚠️ 需手动维护映射表 |
| **编辑能力** | ✅ 内置单元格编辑 | ✅ 优秀（setFlags+delegate） | ❌ 不支持（需弹窗替代） |
| **右键菜单** | ✅ 标准tkinter实现 | ✅ QMenu标准实现 | ✅ 已实现 |
| **开发成本** | 🟢 低（1天Demo验证） | 🔴 高（需引入PyQt5，改写全部GUI） | 🟢 最低（已实现，优化即可） |
| **维护性** | 🟡 中等（社区较小，文档有限） | 🟡 中等（PyQt5文档丰富但API复杂） | 🟢 高（无额外依赖） |
| **打包体积** | ⚠️ +5~10MB（pandas依赖已存在） | 🔴 +30~50MB（PyQt5整体） | 🟢 0（无新增） |
| **杀毒误报** | ⚠️ 低风险（纯Python） | 🔴 中风险（PyQt5 DLL多，易触发） | 🟢 无 |
| **低版本系统适配** | ⚠️ Win7+（依赖pandas/numpy版本） | ⚠️ Win7+（PyQt5 5.15支持Win7） | ✅ 最好（纯tkinter，XP也可） |
| **现有代码改动量** | 🟡 中（需替换Treeview为pandastable控件） | 🔴 巨大（整个GUI框架需重写） | 🟢 最小（渐进优化） |
| **风险** | 🟡 中（性能上限、社区活跃度） | 🔴 高（重写风险、打包体积、误报） | 🟢 低（增量改进） |

## 结论

| 方案 | 推荐度 | 适用场景 |
|------|--------|---------|
| **优化现有 Treeview** | ⭐⭐⭐⭐⭐ | **当前首选**：风险最低，已验证可用，渐进优化 |
| **pandastable** | ⭐⭐⭐ | 备选：如果Treeview优化后仍不满足需求（编辑/颜色粒度） |
| **PyQt5 QTableView** | ⭐ | 不推荐：改动成本远超收益，打包体积和误报风险大 |

## 建议

1. **短期**：继续优化现有 Treeview（冻结行头、虚拟滚动优化、编辑弹窗改进）
2. **中期**：如果用户提出单元格内编辑需求，评估 pandastable 替换方案
3. **长期**：如果有完整GUI重构计划，考虑 PyQt5/PySide6（但需独立项目评估）
