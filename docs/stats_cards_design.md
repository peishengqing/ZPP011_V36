# 📊 审核统计桌面卡片 — 设计方案

## 一、定位

在主界面增加一行统计卡片，分析完成后自动显示，让裴哥**一眼看清这批次数据的关键指标**，不依赖日期、不依赖数据范围。

---

## 二、卡片内容（4张）

```
┌──────────────────────────────────────────────────┐
│  📊 本次分析概览                                   │
│  ┌────────┐ ┌────────┐ ┌────────┐ ┌────────────┐ │
│  │  ✅75% │ │ ⚠️38条 │ │ 🔴 5   │ │ 🔄 3组     │ │
│  │ 合格率 │ │ 需处理 │ │ 真异常 │ │ 抵消 ¥2400 │ │
│  └────────┘ └────────┘ └────────┘ └────────────┘ │
└──────────────────────────────────────────────────┘
```

| 卡片 | 数值 | 颜色 | 算法（精确） |
|------|------|------|-------------|
| 合格率 | 百分比 | 绿 #4caf50 | `audit_result == "合格"` 的记录数 ÷ 总记录数（有 `audit_result` 值的行）。如果全部未审核，显示 "--" 而非 0% |
| 需处理 | 条数 | 橙 #ff9800 | `audit_result` 在 `["需关注", "需改进", "需补备注"]` 中的记录数 + `audit_result` 为空/未审核的记录数 |
| 真异常 | 条数 | 红 #f44336 | 满足 **`是替代料 != "是"`** 且 **`偏差率(%) 的绝对值 > 30`** 的记录数 |
| 替代料 | 组数+金额 | 紫 #9c27b0 | **组数**：唯一 `_替代料组` 非空值的数量  **金额**：`净偏差金额` 求和后再取绝对值（因为同一组的正负抵消后净偏差是各组共享的） |

### 关键设计点
- **"真异常"排除替代料**：替代料的偏差率是抵消前的"假偏差"，剔除后才是真正需要关注的异常行
- **"需处理"包含未审核行**：尚未 AI 审核的记录也算"待处理"，督促裴哥去审完
- **合格率只算已审核**：未审核的不拉低合格率，两者分开看更合理
- **替代料金额展示总和绝对值**：如果3组替代料分别是 +1000、-500、+200，展示 `|1000-500+200| = 700`，即抵消后的净影响

## 三、放置位置

在主表格上方，操作按钮组下方：

```
当前布局：
  进度条 → 操作按钮（分析/AI审核/导出） → 主表格 → 日志

改为：
  进度条 → 操作按钮（分析/AI审核/导出）
    → 📊 统计卡片（新增，分析完成后显示）
    → 主表格 → 日志
```

## 四、交互

| 操作 | 效果 |
|------|------|
| 分析完成 | 卡片自动出现，显示统计数据 |
| 点击"真异常"卡片 | **叠加筛选**：在现有筛选条件之上，追加 `是替代料 != "是" 且 abs(偏差率) > 30` 的过滤，表格只显示真异常行 |
| 再次点击"真异常"卡片（已激活状态） | 清除卡片叠加的筛选，恢复到当前筛选栏状态 |
| 筛选栏条件变化 | 卡片数字联动刷新（始终反映当前可见数据的统计） |
| 鼠标悬停卡片 | 弹出 tooltip 说明算法（如"真异常：非替代料中偏差率>30%的条数"） |
| 无数据时 | 卡片区域隐藏，不占空间 |

### 筛选交互规则

- 卡片点击 = **叠加筛选**，不替换用户已在筛选栏设置的条件
- 卡片激活时，边框高亮（如 `border: 2px solid #f44336`）
- 卡片区域左侧有一个小的"清除筛选"按钮（🔍 样式），点击后移除所有卡片叠加的筛选

## 六、跟现有功能的区别

| | DashboardDialog（已有） | 统计卡片（新增） |
|--|------------------------|-----------------|
| 位置 | 独立弹窗，入口深 | 主界面直接可见 |
| 信息量 | 重，图表+统计全 | 轻，4个核心数字 |
| 使用场景 | 深入分析 | 一眼掌握 |
