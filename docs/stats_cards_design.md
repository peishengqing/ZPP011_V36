明白了，你的工作流是 **AI审完 → 看一眼备注 → 标记已读**，不是"一条条去审核"。

所以卡片不该按"审核结果"来分（合格/需处理那些对你没用），应该按**你实际的操作**来——已读/未读。

```
┌──────────────────────────────────────────────┐
│  📊 本次概览                                  │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌───────────┐  │
│  │ ✅75%│ │ 📖32条│ │ 🔴 5 │ │ 🔄 3组   │  │
│  │ AI通过│ │ 未读  │ │ 真异常│ │ 抵消¥2400│  │
│  └──────┘ └──────┘ └──────┘ └───────────┘  │
```

| 卡片 | 含义 | 数据来源 |
|------|------|---------|
| **AI通过** | AI审核为"合格"的占比 | `audit_result` 字段 |
| **未读** | 你还没看过的条数 | `read_status` 已读状态 |
| **真异常** | 非替代料中偏差率>30%的条数 | 直接从数据算 |
| **替代料** | 组数+抵消金额 | 替代料相关列 |

"AI通过"告诉你这批次质量怎么样，"未读"告诉你还有多少没看完。这两个才是你真正关心的数字。其他两张卡片保持原样。
