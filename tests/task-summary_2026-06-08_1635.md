# 单元测试创建 — 已完成

## 目标
为 `core/fingerprint.py`、`core/read_status.py`、`analysis/analyzer.py`、`analysis/net_offset.py`、`gui_pyside6/services/data_service.py` 核心函数编写单元测试。

## 成果
- **5 个测试文件，29 个测试用例，全部通过**
- 使用 pytest + pytest-cov，覆盖率报告已生成到 `htmlcov/`
- 核心测试覆盖：
  - `_build_deviation_summary` — 分组汇总、排序、缺失列处理
  - `apply_net_offset` — 抵消计算、enable=False、空配对、group_key 显式传入
  - `DataService.preprocess_audit_data` — data_id 生成（含/不含工厂）、fingerprint 计算、去重
  - `DataService._normalize_alt_flag` — 标准化标记、缺失列填充
  - `calc_fingerprint` — 确定性校验、不同输入区分、负值/零值/大数值
  - `save_read_status` / `load_read_status` — 读写、覆盖、批量、空列表

## 注意事项
- `do_analysis_v2` 依赖 20+ 个特定列名的 Excel 文件且强耦合 pandas I/O，不适合单元测试，未覆盖
- `infer_material_type` 是嵌套函数（定义在 `do_analysis_v2` 内部），不可直接 import 测试
- s01_test_data.xlsx 列名含空格差异（如"数量 - 定额"），与代码预期不符
